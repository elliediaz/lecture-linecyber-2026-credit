# 왜 AI 윤리가 중요한가?

## AI 기술의 양면성

| 긍정적 측면 | 부정적 측면 |
|------------|------------|
| 생산성 향상 | 일자리 대체 우려 |
| 의사결정 지원 | 편향된 판단 |
| 반복 업무 자동화 | 책임 소재 불명확 |
| 품질 향상 | 프라이버시 침해 |

> **AI 개발자의 책임**: 기술을 올바르게 사용하여 사회에 긍정적 영향을 미치는 것

---

# AI 윤리 4대 원칙

## 1. 공정성 (Fairness)

- AI 시스템은 **모든 사용자를 공평하게** 대우해야 함
- 성별, 인종, 나이 등에 따른 차별 금지

### 제조 현장 예시
```
품질 검사 AI가 특정 생산 라인의 제품만
더 엄격하게 판정한다면? → 불공정
```

---

# AI 윤리 4대 원칙

## 2. 투명성 (Transparency)

- AI의 **의사결정 과정이 설명 가능**해야 함
- "블랙박스" 문제 해결 필요

### 제조 현장 예시
```
불량 판정 AI가 "왜" 불량이라고 판단했는지
작업자에게 설명할 수 있어야 함
```

---

# AI 윤리 4대 원칙

## 3. 책임성 (Accountability)

- AI 시스템의 결과에 대한 **책임 주체 명확화**
- 문제 발생 시 대응 체계 구축

### 질문
```
자율주행 트럭이 사고를 냈다면, 누구 책임인가?
- 차량 제조사?
- AI 개발사?
- 운전자(관리자)?
- 화물 운송사?
```

---

# AI 윤리 4대 원칙

## 4. 프라이버시 (Privacy)

- 개인정보 및 기업 비밀 **보호 의무**
- 데이터 수집·저장·활용 시 동의 필요

### 제조 현장 예시
```
작업자 동작 분석 AI → 개인 행동 패턴 수집
→ 동의 없이 인사 평가에 활용? → 프라이버시 침해
```

---

# 제조 현장 AI 윤리 사례

## 사례 1: 편향된 품질 검사

### 상황
- AI 품질 검사 시스템 도입
- 특정 작업자가 담당한 제품의 불량률이 높게 나옴

### 문제점
- 조명 조건, 카메라 위치 등 환경 차이로 인한 편향
- 작업자 개인에 대한 부당한 불이익

### 해결책
- 다양한 환경에서 학습 데이터 수집
- 정기적인 편향성 검증

---

# 제조 현장 AI 윤리 사례

## 사례 2: 예지보전과 일자리

### 상황
- AI 예지보전 시스템 도입으로 유지보수 인력 감축 우려
- 현장 작업자들의 반발

### 해결 방향
- AI는 **보조 도구**로 활용
- 작업자를 **AI 관리자/분석가**로 역할 전환
- 교육 훈련 기회 제공

> **핵심**: AI는 사람을 대체하는 것이 아니라 **사람의 역량을 확장**

# 데이터 보안 유형

## 1. 개인정보 보호

### 개인정보보호법 주요 내용
- 수집 시 **명확한 동의** 필요
- 목적 외 사용 금지
- 보유 기간 경과 시 **즉시 파기**
- 위반 시 과태료 최대 5천만원

### AI 개발 시 주의점
```python
# 잘못된 예시 - 개인정보 직접 노출
df = pd.read_csv('employee_data.csv')
print(df['주민등록번호'])  # 절대 금지!

# 올바른 예시 - 비식별화 처리
df['employee_id'] = df['이름'].apply(hash_function)
```

# 데이터 보안 유형

## 3. 접근 권한 관리

### 원칙: 최소 권한 (Least Privilege)
- 업무에 필요한 **최소한의 데이터만** 접근
- 불필요한 권한 부여 금지

### 접근 권한 예시

| 역할 | 접근 가능 데이터 |
|------|----------------|
| 현장 작업자 | 본인 담당 라인 데이터 |
| 품질 관리자 | 전체 품질 데이터 |
| 데이터 분석가 | 비식별화된 분석 데이터 |
| 시스템 관리자 | 전체 시스템 로그 |
