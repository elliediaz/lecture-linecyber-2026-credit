## 실습편 (15-20분)

### 실습 소개 [1.5min]

> 이제 실습 시간입니다. 실제로 데이터를 가져와봅니다.

> 3. 데이터 구조 확인 방법 습득
>
> **실습 환경**을 확인해주세요.
>
> ```python
> import pandas as pd
> import requests
> ```

### 실습 1: 공공데이터포털 API [3min]

> 첫 번째 실습입니다. 공공데이터포털 API를 사용해봅니다.
>
> 먼저 data.go.kr에 회원가입하고 API 키를 발급받아야 해요.
> 오픈 API 메뉴에서 활용 신청하면 됩니다.
>
> ```python
> import requests
>
> API_KEY = "발급받은_키"
> url = "https://api.odcloud.kr/api/..."
> params = {"serviceKey": API_KEY, "page": 1}
>
> response = requests.get(url, params=params)
> data = response.json()
> ```
>
> 이렇게 프로그램에서 직접 데이터를 가져올 수 있어요.

### 실습 2: Kaggle 데이터셋 [3min]

> 두 번째 실습입니다. Kaggle에서 데이터를 다운로드합니다.
>
> kaggle.com에 접속해서 Datasets 메뉴로 가세요.
> 검색창에 "manufacturing" 또는 원하는 키워드를 입력합니다.
>
> 마음에 드는 데이터셋을 선택하고 Download 버튼을 누르면 됩니다.
>
> 더 편하게 하려면 Kaggle API를 사용할 수도 있어요.
>
> ```python
> # 명령어로 다운로드
> # !kaggle datasets download -d dataset-name
>
> # 다운로드 후 로드
> df = pd.read_csv('data.csv')
> print(df.head())
> ```

### 실습 3: UCI 데이터셋 [3min]

> 세 번째 실습입니다. UCI Repository 데이터를 가져옵니다.
>
> ucimlrepo 라이브러리를 사용하면 아주 간단해요.
>
> ```python
> from ucimlrepo import fetch_ucirepo
>
> # Iris 데이터셋 (ID: 53)
> iris = fetch_ucirepo(id=53)
>
> X = iris.data.features
> y = iris.data.targets
>
> print(X.head())
> ```
>
> fetch_ucirepo 함수에 데이터셋 ID만 넣으면 자동으로 가져와요.
> 특성(X)과 타겟(y)이 분리되어 나옵니다.

### 실습 4: 데이터 구조 확인 [3min]

> 네 번째 실습입니다. 데이터를 받으면 반드시 확인할 것들이 있어요.
>
> ```python
> # 1. 데이터 크기
> print(df.shape)
>
> # 2. 컬럼 정보
> print(df.columns)
>
> # 3. 데이터 타입
> print(df.dtypes)
>
> # 4. 결측치 확인
> print(df.isnull().sum())
>
> # 5. 기본 통계
> print(df.describe())
> ```
>
> 이 5가지를 확인하면 데이터가 어떤 상태인지 파악할 수 있어요.
> 지난 시간에 배운 head, info, describe와 연결되는 내용이죠!

### 실습 5: 제조 데이터 예시 [2min]

> 다섯 번째 실습입니다. 제조 데이터 예시를 만들어봅니다.
>
> ```python
> import pandas as pd
>
> data = {
>     'temperature': [85, 87, 92, 88, 90],
>     'humidity': [50, 52, 55, 48, 51],
>     'speed': [100, 102, 98, 105, 101],
>     'defect': [0, 0, 1, 0, 1]
> }
>
> df = pd.DataFrame(data)
> print(df)
> print(df.describe())
> ```
>
> 앞으로 이런 형태의 제조 센서 데이터를 자주 다루게 될 거예요.
