# EDA란?

## Exploratory Data Analysis

> 데이터를 **탐색**하여 패턴, 이상치, 관계를 **발견**하는 과정

### 목적
1. 데이터 이해
2. 가설 수립
3. 모델링 방향 설정
4. 데이터 품질 확인

---

# EDA 단계

## 체계적 접근

```
1. 데이터 로드 및 개요 파악
   ↓
2. 단변량 분석 (각 변수 개별 분석)
   ↓
3. 이변량 분석 (변수 간 관계)
   ↓
4. 다변량 분석 (여러 변수 동시)
   ↓
5. 인사이트 정리 및 가설 수립
```

---

# 1단계: 데이터 개요

## 첫인상 파악

```python
# 기본 정보
df.shape           # 크기
df.dtypes          # 데이터 타입
df.head()          # 샘플 확인
df.info()          # 요약 정보
df.describe()      # 기술통계
df.isnull().sum()  # 결측치
```

---

# 2단계: 단변량 분석

## 각 변수 개별 분석

### 수치형 변수
- 히스토그램: 분포 형태
- 상자그림: 이상치, 범위
- 기술통계: 평균, 중앙값, 표준편차

### 범주형 변수
- 빈도표: value_counts()
- 막대그래프: 범주별 빈도

---

# 3단계: 이변량 분석

## 두 변수 관계

### 수치 vs 수치
- 산점도
- 상관계수

### 범주 vs 수치
- 그룹별 상자그림
- 그룹별 평균 비교

### 범주 vs 범주
- 교차표 (crosstab)
- 히트맵

---

# 4단계: 다변량 분석

## 여러 변수 동시 분석

```python
# 상관행렬
df.corr()

# Pair plot (seaborn)
import seaborn as sns
sns.pairplot(df)

# 피벗 테이블
df.pivot_table(values='불량률',
               index='라인',
               columns='시간대',
               aggfunc='mean')
```

---

# 5단계: 인사이트 도출

## 발견한 패턴 정리

### 좋은 인사이트의 특징
- **구체적**: "라인 B의 불량률이 평균 2%p 높음"
- **실행 가능**: "온도 85도 이상에서 불량 증가 → 온도 관리 강화"
- **데이터 기반**: 숫자와 그래프로 뒷받침

---

# EDA 체크리스트

## 빠뜨리지 않기

- [ ] 결측치 확인 및 처리 방안
- [ ] 이상치 확인 및 처리 방안
- [ ] 각 변수의 분포 확인
- [ ] 타겟 변수와의 관계 확인
- [ ] 변수 간 상관관계 확인
- [ ] 시간적 패턴 확인 (시계열인 경우)
- [ ] 그룹별 차이 확인

# 실습: 제조 데이터 종합 EDA

# EDA 결과 보고서 구조

## 표준 형식

```
1. 분석 개요
   - 데이터 설명, 분석 목적

2. 데이터 품질 확인
   - 결측치, 이상치 현황

3. 주요 발견 사항
   - 핵심 인사이트 (3~5개)
   - 시각화 자료

4. 결론 및 권고사항
   - 실행 가능한 제안
```
