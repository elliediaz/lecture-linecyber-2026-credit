#### 왜 Python인가? [2분]

> 먼저 왜 Python을 배우는지 알아봅시다.
>
> Python은 AI/ML 분야에서 90% 이상 사용되는 표준 언어입니다. 구글, 넷플릭스, 인스타그램 같은 회사들도 Python을 핵심 기술로 사용하고 있죠.
>
> 가장 큰 장점은 **쉽다**는 겁니다. 다른 언어들에 비해 영어 문장처럼 읽히는 직관적인 문법을 가지고 있어요.
>
> 그리고 NumPy, Pandas, TensorFlow 같은 강력한 라이브러리가 풍부합니다. 우리가 직접 다 만들 필요 없이 가져다 쓰면 돼요.

#### Python 기본 자료형 [2분]

> 이제 Python 문법을 배워봅시다. 먼저 **자료형**입니다.
>
> Python에서 꼭 알아야 할 자료형은 5가지입니다.
>
> **정수(int)**: 생산량 1000개, 라인 번호 3번 같은 소수점 없는 숫자
> **실수(float)**: 불량률 0.025, 온도 85.7도 같은 소수점 있는 숫자
> **문자열(str)**: "센서 모듈 A", "정상" 같은 텍스트
> **리스트(list)**: 여러 데이터를 순서대로 담는 자료형. 대괄호로 만들어요.
> **딕셔너리(dict)**: 이름표를 붙여 데이터를 저장. 중괄호로 만들어요.
>
> 이 5가지가 데이터의 기본 단위입니다.

#### 조건문과 반복문 [2분]

> 다음은 **제어문**입니다.
>
> **조건문 if**는 "만약 ~라면 ~해라"를 코드로 표현합니다.
>
> ```python
> if defect_rate > 0.05:
>     print("경고")
> elif defect_rate > 0.02:
>     print("주의")
> else:
>     print("정상")
> ```
>
> if 다음에 조건을 쓰고 콜론을 찍습니다. 그 다음 줄은 들여쓰기를 해야 해요.
>
> **반복문 for**는 같은 작업을 여러 번 실행할 때 사용합니다.
>
> ```python
> for temp in temperatures:
>     print(temp)
> ```
>
> 리스트의 각 요소가 차례로 temp 변수에 들어가면서 반복됩니다.

### 실습 1: 변수와 계산 [3분]

> 첫 번째 실습입니다. Jupyter Notebook에서 새 셀을 만들고 따라해보세요.
>
> ```python
> # 센서 측정값 입력
> temperature = 85.2      # 온도
> pressure = 101.3        # 압력
> humidity = 45           # 습도
>
> # 일 생산량과 불량률
> daily_production = 1200
> defect_rate = 0.025
>
> # 불량 수 계산
> defect_count = daily_production * defect_rate
> good_count = daily_production - defect_count
>
> print(f"일 생산량: {daily_production}개")
> print(f"양품: {good_count}개, 불량: {defect_count}개")
> ```
>
> Shift+Enter를 눌러서 실행해보세요. 결과가 나오나요?
>
> f-string을 사용하면 변수 값을 문자열에 쉽게 넣을 수 있습니다. 문자열 앞에 f를 붙이고 중괄호 안에 변수를 넣으면 됩니다.

### 실습 2: 조건문으로 품질 판정 [4분]

> 두 번째 실습입니다. 불량률에 따라 품질 등급을 판정해봅시다.
>
> ```python
> defect_rate = 0.025  # 2.5%
>
> if defect_rate <= 0.01:
>     grade = "A"
>     message = "우수"
> elif defect_rate <= 0.03:
>     grade = "B"
>     message = "양호"
> elif defect_rate <= 0.05:
>     grade = "C"
>     message = "주의"
> else:
>     grade = "D"
>     message = "개선 필요"
>
> print(f"품질 등급: {grade} ({message})")
> ```
>
> 불량률 2.5%는 0.03 이하이므로 B등급이 나옵니다.
>
> defect_rate 값을 바꿔서 다른 등급이 나오는지 확인해보세요.

### 실습 3: 반복문으로 온도 분석 [4분]

> 세 번째 실습입니다. 1시간 간격으로 측정한 온도 데이터를 분석해봅시다.
>
> ```python
> temperatures = [82, 85, 88, 95, 84, 91, 86]
> threshold = 90  # 경고 기준
>
> print("=== 온도 모니터링 ===")
> for i, temp in enumerate(temperatures):
>     if temp > threshold:
>         print(f"[경고] {i+1}시: {temp}도 - 기준 초과!")
>     else:
>         print(f"[정상] {i+1}시: {temp}도")
>
> # 평균 온도 계산
> average = sum(temperatures) / len(temperatures)
> print(f"\n평균 온도: {average:.1f}도")
> ```
>
> enumerate 함수는 인덱스와 값을 동시에 가져옵니다. i는 인덱스(0부터 시작), temp는 온도값입니다.
>
> 90도를 초과하는 4시와 6시에 경고가 출력됩니다.

### 실습 4: 딕셔너리로 설비 관리 [3분]

> 네 번째 실습입니다. 딕셔너리를 사용해서 설비 상태를 관리해봅시다.
>
> ```python
> equipment = {
>     "EQ-001": {"온도": 82, "진동": 0.3, "상태": "정상"},
>     "EQ-002": {"온도": 91, "진동": 0.8, "상태": "주의"},
>     "EQ-003": {"온도": 85, "진동": 0.4, "상태": "정상"}
> }
>
> print("=== 설비 상태 점검 ===")
> for eq_id, data in equipment.items():
>     temp = data["온도"]
>     status = data["상태"]
>     print(f"{eq_id}: 온도 {temp}도, 상태 {status}")
> ```
>
> 딕셔너리 안에 딕셔너리를 넣을 수 있습니다. 이런 구조를 중첩 딕셔너리라고 해요.
>
> 실무에서 센서 데이터, 설비 정보 같은 걸 다룰 때 이런 구조를 많이 씁니다.

### 실습 5: 종합 예제 [3분]
