#### 슬라이드 39-40: 데이터 생성

> "먼저 실습용 데이터를 만듭니다. 200개의 제조 데이터를 생성합니다."

> "temperature, humidity, speed 세 가지 특성과, defect(불량 여부)와 production(생산량) 두 가지 타겟을 만듭니다."

```python
import numpy as np
import pandas as pd

np.random.seed(42)
n = 200

df = pd.DataFrame({
    'temperature': np.random.normal(85, 5, n),
    'humidity': np.random.normal(50, 10, n),
    'speed': np.random.normal(100, 15, n),
})

# 불량 여부 (분류 타겟)
defect_prob = 0.1 + 0.02 * (df['temperature'] - 80)
df['defect'] = (np.random.random(n) < defect_prob).astype(int)

# 생산량 (회귀 타겟)
df['production'] = 1000 + 5*df['speed'] - 3*df['temperature'] + \
                   np.random.normal(0, 50, n)

print(df.head())
```

> "defect는 0 또는 1, production은 연속 숫자인 걸 확인하세요."

---

#### 슬라이드 41-42: 특성/타겟 분리 및 데이터 분리

> "X는 특성(입력), y는 타겟(출력)입니다."

```python
X = df[['temperature', 'humidity', 'speed']]
y_clf = df['defect']  # 분류용
y_reg = df['production']  # 회귀용
```

> "train_test_split으로 80:20으로 나눕니다."

```python
from sklearn.model_selection import train_test_split

X_train, X_test, y_train_clf, y_test_clf = train_test_split(
    X, y_clf, test_size=0.2, random_state=42
)
```

---

#### 슬라이드 43-44: 분류 모델 실습

> "먼저 분류 모델입니다. 의사결정나무를 사용합니다."

```python
from sklearn.tree import DecisionTreeClassifier

# 모델 생성 및 학습
model_clf = DecisionTreeClassifier(random_state=42)
model_clf.fit(X_train, y_train_clf)

# 예측 및 평가
y_pred_clf = model_clf.predict(X_test)
accuracy = model_clf.score(X_test, y_test_clf)
print(f"분류 정확도: {accuracy:.1%}")
```

> "fit으로 학습하고, predict로 예측하고, score로 평가합니다. 이 패턴을 기억하세요!"

---

#### 슬라이드 45-46: 회귀 모델 실습

> "이번엔 회귀 모델입니다. 선형회귀를 사용합니다."

```python
from sklearn.linear_model import LinearRegression

model_reg = LinearRegression()
model_reg.fit(X_train, y_train_reg)

y_pred_reg = model_reg.predict(X_test)
r2 = model_reg.score(X_test, y_test_reg)
print(f"R² 점수: {r2:.3f}")
```

> "분류든 회귀든 패턴이 똑같죠? fit, predict, score. 이게 sklearn의 큰 장점입니다."

#### 슬라이드 47-48: 새 데이터 예측

> "학습된 모델로 새 데이터를 예측해봅시다."

```python
# 새로운 제품 데이터
new_product = pd.DataFrame({
    'temperature': [87],
    'humidity': [55],
    'speed': [105]
})

# 분류: 불량 여부
defect_pred = model_clf.predict(new_product)
print(f"불량 예측: {'불량' if defect_pred[0] == 1 else '정상'}")

# 회귀: 생산량
prod_pred = model_reg.predict(new_product)
print(f"생산량 예측: {prod_pred[0]:.0f}개")
```

> "실제 현장에서는 이렇게 센서에서 실시간으로 들어오는 데이터로 예측할 수 있습니다."

## ❓ 예상 질문 및 답변

### Q1: 딥러닝도 머신러닝인가요?

> "네, 맞습니다. 딥러닝은 머신러닝의 한 종류입니다. 신경망이라는 특별한 구조를 사용하는 머신러닝이죠. 19-21차시에서 딥러닝을 따로 배웁니다."

### Q2: 데이터가 몇 개 이상 있어야 머신러닝이 가능한가요?

> "정해진 숫자는 없지만, 일반적으로 특성 수의 10배 이상은 필요합니다. 특성이 3개면 최소 30개, 많으면 많을수록 좋습니다. 복잡한 패턴을 학습하려면 수천~수만 개가 필요할 수도 있어요."

### Q3: 왜 random_state=42를 쓰나요?

> "재현성 때문입니다. random_state를 고정하면 같은 결과가 나와서 디버깅이나 결과 공유가 쉬워요. 42는 특별한 의미는 없고, 관례적으로 많이 쓰는 숫자입니다."

### Q4: 회귀에서 R² 점수가 뭔가요?

> "모델이 데이터를 얼마나 잘 설명하는지 나타내는 지표입니다. 1에 가까울수록 좋고, 0이면 평균으로 예측하는 것과 같은 수준입니다. 자세한 건 15차시 '모델 평가'에서 다룹니다."

### Q5: 분류와 회귀 중 어떤 게 더 어렵나요?

> "문제에 따라 다릅니다. 일반적으로 클래스 불균형이 심한 분류(예: 불량률 1%)는 어렵고, 비선형적인 관계를 예측하는 회귀도 어려울 수 있습니다. 어떤 게 더 어렵다고 단정할 순 없어요."
