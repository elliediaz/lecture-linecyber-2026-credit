## 예상 질문 및 답변

### Q1: EDA에 정해진 순서가 있나요?

**A:** 5단계는 **권장 순서**이지, 절대적인 규칙은 아닙니다. 다만:
- 1단계(개요)는 반드시 먼저
- 2단계(단변량)와 3단계(이변량)는 동시에 해도 됨
- 5단계(인사이트)는 마지막에

분석하다 보면 다시 앞 단계로 돌아가기도 합니다. **반복적 과정**이에요.

### Q2: 인사이트를 몇 개 정도 도출해야 하나요?

**A:** 분석 목적과 데이터에 따라 다르지만, 일반적으로 **3~5개의 핵심 인사이트**를 정리합니다:
- 너무 적으면: 분석이 부족한 것
- 너무 많으면: 우선순위가 없는 것

**우선순위**를 매겨서 가장 중요한 것 위주로 정리하세요.

### Q3: 시각화를 꼭 해야 하나요?

**A:** **반드시** 해야 합니다. 이유:
1. 숫자만으로는 놓치는 패턴이 있음
2. 이상치가 시각적으로 더 잘 보임
3. 커뮤니케이션에 필수
4. 가설 수립에 도움

"그림으로 보여줘야 사람들이 이해합니다."

### Q4: 상관계수가 낮으면 의미 없는 건가요?

**A:** 꼭 그렇지는 않습니다:
- **비선형 관계**일 수 있음 (산점도 확인!)
- **조건부 상관**이 있을 수 있음
- **다른 변수와의 조합**에서 의미 있을 수 있음

상관계수가 낮아도 **산점도를 반드시 확인**하세요.

### Q5: EDA 결과를 어떻게 보고하나요?

**A:** 표준 형식을 권장합니다:
1. **분석 개요**: 목적, 데이터 설명
2. **데이터 품질**: 결측치, 이상치 현황
3. **주요 발견 사항**: 인사이트 3~5개 + 시각화
4. **결론 및 권고**: 실행 가능한 제안
